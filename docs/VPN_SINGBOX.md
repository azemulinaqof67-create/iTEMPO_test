# Руководство по VPN (sing-box) и обходу блокировок

Данный документ подробно описывает механизм обхода региональных блокировок для сервисов ИИ (Google Gemini API), используемый на сервере Ubuntu, а также инструменты автоматического мониторинга.

## 1. Архитектура решения

Сервисы Google (включая API моделей Gemini) ограничивают доступ для IP-адресов из РФ. Чтобы ИИ-бот мог беспрепятственно отправлять запросы, весь интернет-трафик сервера заворачивается в зарубежный туннель.

В качестве клиента используется **sing-box** (ядро, на котором базируется популярный клиент Hiddify).

### Как это работает "под капотом":
1. При запуске служба `sing-box.service` создает виртуальный сетевой интерфейс `tun0`.
2. Ядро Linux переписывает таблицу маршрутизации (Routing Table) таким образом, что все исходящие запросы в интернет (в том числе от Docker-контейнеров) направляются в этот виртуальный интерфейс.
3. `sing-box` перехватывает эти пакеты (а также DNS-запросы), шифрует их и отправляет на удаленный зарубежный прокси-сервер (VLESS, Shadowsocks и т.д.).
4. Удаленный сервер выполняет запрос к Google от своего имени и возвращает ответ обратно в туннель.
5. Для бота это выглядит так, будто сервер физически находится в Европе (например, в Латвии).

## 2. Управление службой sing-box

Конфигурационные файлы ядра sing-box находятся в директории `/etc/sing-box/`. Основной файл настроек — `config.json`.

**Основные команды для управления:**
- **Посмотреть статус:** `sudo systemctl status sing-box`
- **Прочитать логи ядра:** `sudo journalctl -u sing-box -n 50 --no-pager`
- **Перезапустить VPN:** `sudo systemctl restart sing-box`
- **Остановить VPN:** `sudo systemctl stop sing-box`

### Проверка работоспособности туннеля
Чтобы убедиться, что трафик контейнера бота действительно идет через зарубежный сервер, выполните проверку IP-адреса изнутри Docker-контейнера:
```bash
sudo docker exec tempo-app curl -s ipinfo.io/country
```
*Если в ответ вернётся `RU`, значит туннель не работает (трафик идет напрямую) и запросы к ИИ будут падать с ошибками 503 или 404.*

## 3. Проблема "Зависания" и Watchdog

### Почему VPN зависает?
Периодически системы ТСПУ (системы провайдеров РФ) блокируют узлы или сбрасывают TCP-пакеты "в пустоту" (Silent Drop) без разрыва соединения. 
В такой ситуации системная служба `sing-box` остается запущенной (`active (running)`), но интернет внутри туннеля пропадает. Менеджер процессов `systemd` не перезапускает службу, так как процесс фактически не "упал" (он просто ждет ответа от сети).

### Скрипт авто-восстановления (Watchdog)
Для решения этой проблемы был написан скрипт-лекарь `scripts/vpn_watchdog.sh`.

**Логика работы скрипта:**
1. Запрашивает страну текущего IP-адреса сервера через сервис `ipinfo.io`.
2. Если запрос **зависает** (истекает 10-секундный таймаут) или возвращает **RU** (Россия).
3. Скрипт понимает, что туннель "пробит" или завис, и принудительно вызывает `sudo systemctl restart sing-box`.
4. Все сбои логируются в файл `/var/log/vpn_watchdog.log`.

**Установка скрипта в расписание:**
Скрипт должен быть добавлен в планировщик задач `cron` от имени суперпользователя (`root`), чтобы выполняться каждую минуту:
```bash
# Скопировать скрипт в системную папку
sudo cp scripts/vpn_watchdog.sh /usr/local/bin/vpn_watchdog.sh
sudo chmod +x /usr/local/bin/vpn_watchdog.sh

# Добавить в crontab (sudo crontab -e)
* * * * * /usr/local/bin/vpn_watchdog.sh
```

## 4. Смена прокси-узла (Если сервер навсегда заблокирован)

Если скрипт Watchdog постоянно перезапускает VPN, но он так и не начинает работать, это значит, что IP-адрес зарубежного прокси-сервера был полностью заблокирован либо провайдером РФ, либо самим Google (например, защита от спама).

**Что делать:**
1. Зайдите в панель управления VPN-провайдера (или Hiddify).
2. Сгенерируйте новую ссылку-подписку или скопируйте конфигурацию нового узла.
3. Обновите конфигурацию `sing-box` (либо вставив новый VLESS-ключ в Hiddify, либо обновив файл `/etc/sing-box/config.json`).
4. Выполните рестарт службы: `sudo systemctl restart sing-box`

> **Важно:** Старайтесь не подключаться к этому же VPN-профилю с личного компьютера (Windows/телефон) одновременно с сервером. Многие провайдеры блокируют одновременные сессии с одного ключа, что приведет к немедленному отключению VPN на сервере.
