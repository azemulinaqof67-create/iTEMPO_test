## 1. Резюме архитектуры системы

Для обеспечения бесперебойной работы корпоративного ИИ-бота в условиях блокировок зарубежных API (OpenAI, Gemini, Anthropic) архитектура системы разделена на два независимых слоя:
1. **Контейнеризация (Docker):** Обеспечивает изолированную работу кода приложения бота и его базы данных.
2. **Сетевой прокси-туннель (sing-box / Hiddify):** Глобальная системная служба на хост-сервере. Она перехватывает трафик системы и контейнеров, безопасно перенаправляя его через зарубежный узел.

Внутри контейнера `tempo-app` запущен один изолированный мастер-процесс:

Скрипт отвечает за инициализацию и асинхронное выполнение всех внутренних подмодулей ИИ-ботов компании.

## 3. Механизм обхода ограничений (VPN/Proxy)

Внутри самих Docker-контейнеров параметры прокси отсутствуют. Используется схема **прозрачного проксирования всего сервера**.

На хост-машине Ubuntu развернуто ядро **sing-box** (база клиента **Hiddify**), работающее как системная служба:

    /usr/bin/sing-box -D /var/lib/sing-box -C /etc/sing-box run

### Сетевой интерфейс TUN
При запуске `sing-box` создает виртуальную сетевую карту уровня ядра:

    3: tun0: <POINTOPOINT,MULTICAST,NOARP,UP,LOWER_UP> mtu 9000
        inet 172.19.0.1/30 brd 172.19.0.3 scope global tun0

> **Принцип работы TUN-режима:** > Весь исходящий интернет-трафик сервера Ubuntu, включая сетевой мост Docker, перехватывается таблицей маршрутизации и принудительно направляется в интерфейс `tun0`. Ядро `sing-box` шифрует эти пакеты и передает их на удаленный зарубежный прокси-сервер. Поэтому новые контейнеры и проекты автоматически получают доступ к ИИ без дополнительных настроек.

## 4. Верификация сетевой доступности

Тестовый HTTP-запрос к независимому серверу идентификации IP-адресов непосредственно из контекста выполнения контейнера `tempo-app`:

    sudo docker exec tempo-app python -c "import urllib.request; print(urllib.request.urlopen('[https://ifconfig.me](https://ifconfig.me)').read().decode())"

**Фактический результат:** `37.128.205.211` (Внешний адрес выходного зарубежного прокси-сервера). Данный IP-адрес подтверждает, что бот успешно функционирует в обход ограничений.

## 5. Шпаргалка для администратора (Полезные команды)

| Задача | Команда в терминале |
| :--- | :--- |
| **Посмотреть живые логи бота** | `sudo docker logs -f --tail 50 tempo-app` |
| **Перезапустить бота** | `sudo docker restart tempo-app` |
| **Проверить статус VPN-модуля** | `sudo systemctl status sing-box` |
| **Перезапустить VPN-модуль** | `sudo systemctl restart sing-box` |
| **Посмотреть загрузку ресурсов** | `sudo docker stats --no-stream` |

> ⚠️ **Важное примечание:** Конфигурационные файлы прокси находятся по пути `/etc/sing-box/`. Без необходимости не редактируйте файлы в этой директории, чтобы не нарушить общую маршрутизацию сервера.
