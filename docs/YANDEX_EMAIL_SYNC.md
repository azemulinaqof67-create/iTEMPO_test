# Синхронизация почтовых адресов с Яндекс 360

Этот документ описывает работу механизма синхронизации email-адресов контактов в телефонном справочнике с базой данных пользователей организации в сервисе **Яндекс 360**.

---

## 1. Назначение
В локальном телефонном справочнике (`contacts.db`) хранятся ФИО, должности и телефонные номера сотрудников. Поскольку почтовые адреса сотрудников централизованно ведутся в организации Яндекс 360, разработан скрипт, который в автоматическом или ручном режиме обогащает локальную базу контактов актуальными email-адресами сотрудников из Яндекса на основе сопоставления ФИО.

---

## 2. Конфигурация (.env)
Для работы синхронизации в файле `.env` должны быть настроены следующие параметры:

```ini
# OAuth-токен Яндекс 360 с правами на чтение директории (directory:read_users)
YANDEX_360_TOKEN=y0_AgAAA...

# ID организаций в Яндекс 360. Поддерживается перечисление нескольких организаций на новых строках.
# Строки могут содержать комментарии, начинающиеся с символа '#'
YANDEX_360_ORG_ID=123456 # ООО "Компания 1"
                  789012 # ООО "Компания 2"
```

### 2.1. Инструкция по получению OAuth-токена Яндекс 360

Для генерации токена администратору организации необходимо:
1. Зайти в Яндекс под учетной записью администратора организации 360 и перейти к созданию OAuth-приложения: [https://oauth.yandex.ru/client/new](https://oauth.yandex.ru/client/new).
2. Заполнить форму:
   * **Название**: `iTEMPO Contacts Sync`
   * **Платформы**: Отметить чекбокс **«Веб-сервисы»**.
   * **Callback URL**: Нажать ссылку «Подставить URL для отладки» или вручную вписать `https://oauth.yandex.ru/verification_code`.
3. **Доступы (Права)**:
   * В списке прав найти категорию **«Яндекс 360 (Directory API)»**.
   * Развернуть её и отметить галочкой пункт **`directory:read_users`** (Доступ на чтение информации о пользователях).
4. Нажать кнопку **«Создать приложение»** внизу страницы.
5. На странице созданного приложения скопировать **`ID`** (Client ID) приложения (набор букв и цифр).
6. Вставить скопированный ID в шаблон ссылки ниже и открыть её в новой вкладке браузера:
   `https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID`
7. Разрешить приложению доступ к данным.
8. Страница перенаправит вас на экран успешной авторизации, а в адресной строке появится готовый токен. Скопируйте значение токена (начинается с `y0_AgAAA...`).
9. Вставьте его в значение `YANDEX_360_TOKEN` файла `.env`.

## 3. Алгоритм работы
Скрипт синхронизации расположен по пути: [scripts/sync_yandex_emails.py](file:///e:/Old/bots/Worker/iTEMPO/iTEMPO_test/scripts/sync_yandex_emails.py). Процесс состоит из четырёх этапов:

### Этап 1. Запрос к Яндекс 360 API
Скрипт постранично (порциями по 1000 пользователей) запрашивает всех пользователей для каждого указанного ID организации через метод API:
`GET https://api360.yandex.net/directory/v1/org/{org_id}/users`
В заголовке запроса передается OAuth-токен: `Authorization: OAuth {token}`.

### Этап 2. Нормализация и построение карты (ФИО → Email)
Для сопоставления используется функция нормализации ФИО `normalize_name()`, которая:
1. Приводит весь текст к нижнему регистру.
2. Удаляет пробелы в начале и в конце строки.
3. Объединяет множественные пробелы внутри строки в один.

На основе полученных из Яндекса данных строится хэш-карта в памяти. Для каждого аккаунта регистрируется до трёх нормализованных вариантов ключа:
* **По отображаемому имени (`displayName`)**: например, `«Иванов Иван Иванович»`
* **По полному ФИО**: склеиваются поля `last` (Фамилия), `first` (Имя) и `middle` (Отчество) $\rightarrow$ `«Иванов Иван Иванович»`
* **По краткому имени**: склеиваются только поля `last` (Фамилия) и `first` (Имя) $\rightarrow$ `«Иванов Иван»`

Каждый из этих ключей указывает на email-адрес пользователя в Яндексе.

### Этап 3. Сопоставление с локальной БД
Скрипт подключается к базе данных SQLite `data/contacts.db`, включает безопасный режим WAL (`PRAGMA journal_mode=WAL;`) и считывает текущую таблицу контактов:
`SELECT id, full_name, email FROM contacts`

Для каждой строки:
1. ФИО из базы данных (`full_name`) нормализуется с помощью `normalize_name()`.
2. Очищенное имя ищется в карте Яндекса.
3. Если соответствие найдено, и почта из Яндекса **отличается** от текущей почты в локальной БД (или поле почты пустое), контакт добавляется в очередь на обновление.

### Этап 4. Сохранение результатов
Все подготовленные обновления записываются в базу данных пачкой с помощью эффективного метода `executemany`:
```sql
UPDATE contacts SET email = ? WHERE id = ?
```

## 4. Способы запуска

### Ручной запуск из панели администратора
На странице **«Телефонный справочник»** в правом верхнем углу для администраторов с правами редактирования доступна кнопка **«Импорт из Яндекс 360»**.
* При клике отправляется POST-запрос на `/api/contacts/sync_yandex`.
* Веб-интерфейс отображает анимацию загрузки и по завершении выводит тост-уведомление с количеством обновленных контактов.

### Ручной запуск через консоль
Синхронизацию можно запустить вручную напрямую на сервере с помощью команды:
```bash
uv run python scripts/sync_yandex_emails.py
```
Скрипт выведет детальные логи хода выполнения и статистику в стандартный поток вывода.

### Автоматический запуск (Крон / Планировщик задач)
Для поддержания почтовых адресов в актуальном состоянии рекомендуется настроить ежедневный запуск скрипта через планировщик Windows Task Scheduler или cron:
```bash
# Пример для Linux cron (каждую ночь в 3:00)
0 3 * * * cd /path/to/iTEMPO && uv run python scripts/sync_yandex_emails.py >> logs/yandex_sync.log 2>&1
```

## 5. Логирование и диагностика
Ход выполнения синхронизации записывается в стандартные логи приложения. 

**Примеры частых ошибок и способы их решения:**
* `Ошибка запроса к Яндекс 360 API (HTTP 401)`: Указан неверный или просроченный `YANDEX_360_TOKEN`. Перевыпустите OAuth-токен Яндекс 360.
* `Ошибка запроса к Яндекс 360 API (HTTP 403)`: У токена отсутствуют права на чтение справочника организации.
* `Параметры YANDEX_360_TOKEN или YANDEX_360_ORG_ID не заданы в .env`: Убедитесь, что переменные окружения заполнены и файл `.env` находится в корневой директории проекта.
