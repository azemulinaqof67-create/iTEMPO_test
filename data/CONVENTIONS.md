# Соглашения по структуре документов базы знаний ГК ТЭМПО

Этот файл фиксирует шаблоны и правила оформления документов в директории `data/`.
Соблюдение шаблонов обязательно — это обеспечивает консистентность базы знаний и корректную индексацию RAG.

---

## Общие правила

- Каждый файл описывает **одну сущность** (здание, процесс, отдел и т.д.).
- Каждый файл содержит **YAML frontmatter** с обязательными полями.
- Имена файлов — строчные буквы, слова через `_` (snake_case), на английском: `abk_2.md`, `production.md`.
- Для группы связанных файлов создаётся директория с `index.md` — навигационной точкой входа.
- Файл `index.md` не содержит детальных данных — только оглавление и краткое описание дочерних файлов.

---

## YAML Frontmatter

Обязательные поля для всех документов:

```yaml
---
title: "Краткое название документа"
description: "Одно предложение — что описывает файл"
tags: ["тег1", "тег2"]        # ключевые слова для RAG-поиска
questions_answered:            # вопросы, на которые отвечает документ
  - "Где находится ...?"
  - "Что такое ...?"
department: "Company"          # категория: Company / HR / IT и т.д.
company: "ГК «ТЭМПО»"
last_updated: "YYYY-MM-DD"    # дата последнего обновления содержимого
---
```

> `last_updated` обновляется при каждом изменении **содержимого** (не структуры файла).

## Шаблон: Файл здания / объекта

Используется для: `abk_2.md`, `abk_3.md` и любых будущих зданий (склады, производственные корпуса).

```markdown
# Полное название здания (Аббревиатура)

- **Адрес:** ул. Название, N
- **Карта:** [Яндекс.Карты — Название](https://...)

## Инфраструктура и общие службы
- Служба / пункт 1
- Служба / пункт 2

## Размещение по компаниям

| Предприятие | Расположение | Подразделения |
|-------------|--------------|---------------|
| АО «...» | 3-й этаж, левое крыло | Отдел 1, Отдел 2 |
| ООО «...» | 2-й этаж | Отдел 3 |

## Справочник кабинетов
_(раздел добавляется только при наличии данных о конкретных номерах кабинетов)_

| Кабинет | Подразделение |
|---------|---------------|
| 201 | Приемная |
```

### Правила секций здания

| Секция | Обязательна | Когда добавлять |
|--------|-------------|-----------------|
| `## Инфраструктура и общие службы` | Да | Всегда (если нет данных — оставить пустым списком) |
| `## Размещение по компаниям` | Да | Всегда |
| `## Справочник кабинетов` | Нет | Только при наличии данных о номерах кабинетов |

### Правила таблицы «Размещение по компаниям»

- **3 колонки:** `Предприятие` | `Расположение` | `Подразделения`
- Если расположение неизвестно — ставить `—`
- Одно предприятие может занимать несколько строк (разные этажи/крылья)

## Шаблон: Производственные площадки

Используется для: `production.md` и аналогичных файлов.

```markdown
# Производственные площадки и склады

Краткое вводное предложение.

## АО / ООО «Название предприятия»
- **Объект 1:** [Яндекс.Карты](https://...)
- **Объект 2:** [Яндекс.Карты](https://...)
```

## Шаблон: Навигационный index.md

```markdown
# Заголовок раздела

Одно вводное предложение — что содержит раздел.

- **[[Название|filename]]**: Краткое описание.
- **[[Название|filename]]**: Краткое описание.

> **См. также:** [[Смежный документ 1]], [[Смежный документ 2]]
```
